# 语雀类在线文档产品核心功能设计文档

## 1. 文档目标

### 1.1 产品定位
打造一套面向个人、团队和企业场景的在线文档与知识库产品，用于沉淀结构化知识、多人协作编辑、统一检索与内容分发。

### 1.2 核心价值
- 让文档沉淀有结构：通过知识库、目录树、标签、搜索建立统一知识入口。
- 让协作过程可控：通过权限、评论、版本、审计保证多人协作不失控。
- 让内容生产高效：通过编辑器、模板、导入导出、快捷操作提升写作效率。
- 让内容可复用可开放：通过 API、嵌入、分享、导出支持内容流转。

### 1.3 目标用户
- 个人用户：个人笔记、学习记录、项目资料归档。
- 小团队：产品文档、研发文档、运营手册、知识共享。
- 企业组织：制度文档、流程文档、内部协作知识库、开放文档中心。

## 2. 用户角色与权限模型

### 2.1 用户角色
- 平台访客：可访问公开内容，无编辑能力。
- 注册用户：可创建个人知识库，参与被授权协作。
- 团队成员：可访问所属团队空间，根据角色进行协作。
- 团队管理员：负责团队成员、知识库、权限与审计管理。
- 超级管理员：负责平台级配置、风控、用户治理与系统监控。

### 2.2 权限层级
- 平台级权限：用户封禁、全局配置、存储策略、审计查看。
- 团队级权限：成员管理、角色配置、团队空间设置。
- 知识库级权限：可见范围、成员授权、分享策略、导出策略。
- 文档级权限：查看、评论、编辑、管理、复制、导出。

### 2.3 权限策略
- 支持角色继承：文档默认继承知识库权限，可按文档单独覆盖。
- 支持最小授权：默认无权限，需显式授予访问或编辑能力。
- 支持分享控制：公开链接、密码访问、有效期、禁止下载。
- 支持审计留痕：关键操作需要记录操作者、时间、对象、结果。

## 4. 核心功能设计

### 4.1 文档管理

#### 4.1.1 文档结构
- 支持多级目录树：目录、子目录、文档。
- 文档元信息包括标题、摘要、作者、标签、封面、状态、创建时间、更新时间。
- 文档支持草稿态、已发布态、归档态、已删除态。

#### 4.1.2 基本操作
- 新建、编辑、保存、发布、删除、恢复、移动、复制、重命名。
- 支持拖拽排序、批量移动、批量删除、批量标签管理。
- 支持自动保存与手动发布，降低误操作损失。
- 支持文档链接引用、跨文档跳转、面包屑定位。

#### 4.1.3 文档生命周期
- 草稿：作者编辑中，仅本人或授权成员可见。
- 已发布：对授权范围内用户可见。
- 已归档：不再参与默认导航与推荐，但可检索。
- 回收站：软删除，支持恢复与定期清理。

#### 4.1.4 富文本与内容块能力
- 所见即所得编辑器与 Markdown 模式并存。
- 支持标题、段落、列表、引用、分割线、表格、代码块、任务列表。
- 支持插入图片、附件、音视频、链接、卡片、流程图、思维导图。
- 支持代码高亮、复制代码、语言类型识别。
- 支持内容块拖拽排序与局部样式配置。

#### 4.1.5 模板能力
- 提供空白模板、会议纪要、需求文档、技术方案、周报模板。
- 支持团队自定义模板。
- 支持通过模板创建文档时自动填充固定字段。

### 4.2 知识库与空间管理

#### 4.2.1 知识库能力
- 用户可创建多个知识库，按业务、团队、项目划分内容。
- 支持知识库封面、简介、标识、可见范围、导航配置。
- 支持知识库归档、恢复、转移归属、删除。

#### 4.2.2 空间视图
- 支持个人空间、团队空间、公开空间。
- 支持首页推荐、置顶文档、最近更新、热门文档。
- 支持自定义导航入口与知识库首页布局。

### 4.3 协作与沟通

#### 4.3.1 协作编辑
- 首版支持非实时协作：多人可先后编辑，依赖版本与冲突提示。
- 后续支持实时协同：展示在线成员、光标位置、实时变更。
- 支持编辑锁或冲突检测，避免覆盖写入。

#### 4.3.2 评论与讨论
- 支持文档评论、段落评论、回复、@成员。
- 支持评论状态管理：待处理、已解决。
- 支持将评论消息同步到通知中心。

#### 4.3.3 任务协同
- 支持在文档中创建任务项。
- 支持指派负责人、截止时间、完成状态。
- 支持基于 @ 的待办提醒。

### 4.4 权限与分享

#### 4.4.1 可见性策略
- 公开：任何人可访问。
- 组织内可见：团队成员可访问。
- 指定成员可见：仅授权用户可访问。
- 私密：仅作者或管理员可访问。

#### 4.4.2 操作权限
- 查看
- 评论
- 编辑
- 管理
- 导出
- 分享

#### 4.4.3 分享能力
- 生成分享链接。
- 支持密码访问、有效期、访问次数限制。
- 支持关闭评论、禁止复制、禁止导出。

### 4.5 搜索与导航

#### 4.5.1 搜索范围
- 全局搜索：跨知识库搜索标题、正文、标签、作者。
- 知识库内搜索：限定单一知识库。
- 文档内搜索：编辑态与阅读态均支持。

#### 4.5.2 搜索能力
- 支持关键词高亮、分词检索、模糊匹配。
- 支持按更新时间、创建人、标签、知识库筛选。
- 支持搜索历史与热门搜索。

#### 4.5.3 导航能力
- 目录树导航。
- 面包屑导航。
- 最近访问、收藏、我的协作、待我处理。

### 4.6 版本与审计

#### 4.6.1 版本管理
- 记录每次发布版本与关键保存快照。
- 支持查看历史版本、比较差异、恢复指定版本。
- 支持记录版本说明。

#### 4.6.2 操作日志
- 记录文档创建、编辑、删除、恢复、移动、导出、分享等关键操作。
- 记录知识库授权、成员变更、角色调整。
- 日志支持按时间、操作者、对象筛选。

### 4.7 通知与消息

#### 4.7.1 通知触发场景
- 评论与回复。
- @ 提及。
- 权限变更。
- 文档分享。
- 协作任务指派。

#### 4.7.2 通知渠道
- 站内通知。
- 邮件通知。
- 后续可扩展企业微信、钉钉、飞书机器人。

### 4.8 账户与团队

#### 4.8.1 用户体系
- 用户注册、登录、退出、找回密码。
- 支持邮箱登录、手机号登录、第三方登录扩展位。
- 支持头像、昵称、个人简介、语言偏好设置。

#### 4.8.2 团队管理
- 创建团队、邀请成员、移除成员、分配角色。
- 支持按部门或用户组授权。
- 支持团队品牌配置，如名称、Logo、域名映射。

### 4.9 内容导入导出与开放能力

#### 4.9.1 导入能力
- 导入 Markdown 文件。
- 导入 Word 文档。
- 批量导入本地目录。

#### 4.9.2 导出能力
- 导出为 Markdown、PDF、Word、HTML。
- 支持单文档导出与知识库批量导出。

#### 4.9.3 开放 API
- 提供认证接口。
- 提供知识库、文档、评论、用户基础接口。
- 为第三方系统预留 Webhook 扩展能力。

### 4.10 数据统计与运营能力

#### 4.10.1 基础统计
- 文档阅读量、访客数、收藏数、评论数。
- 知识库活跃度、成员贡献度、最近更新趋势。

#### 4.10.2 运营能力
- 热门文档推荐。
- 失效文档识别，如长期未更新文档。
- 内容质量治理，如空文档、重复文档提醒。

## 5. 非功能要求

### 5.1 性能要求
- 常规文档打开时间目标小于 2 秒。
- 文档保存操作应具备秒级反馈。
- 搜索结果首屏返回时间目标小于 2 秒。

### 5.2 安全要求
- 账号密码加密存储。
- 敏感操作二次校验。
- 分享链接防暴力访问与过期失效。
- 关键接口具备频控、防刷、审计能力。

### 5.3 可靠性要求
- 文档自动保存，避免编辑内容丢失。
- 版本快照可回滚。
- 附件存储具备容灾和备份策略。

### 5.4 兼容性要求
- 支持桌面端主流浏览器。
- 移动端优先保证阅读与轻编辑。
- 后续可扩展独立 App 或小程序。

## 7. 关键设计原则

- 以知识沉淀为核心，不以单次编辑为核心。
- 以权限与版本保证协作安全。
- 首版优先做强文档、知识库、权限、搜索四个底座能力。
- 设计上保持模块化，便于未来扩展实时协同、开放平台和企业能力。
- 优先保证稳定与易用，再逐步扩展高级能力。
